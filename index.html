A research-lab walkthrough for humans

What karpathy/autoresearch is, and how to actually run it.

autoresearch is a tiny repo designed for autonomous model experimentation: you prepare data once, let an agent edit train.py, and compare every run on the same five-minute budget.

The repo in one glance

Autoresearch is less “framework” and more “tight experiment contract.”

01

You program the research org in Markdown.

The human mostly shapes program.md: the rules, goals, output format, and experiment loop that the coding agent follows.

02

The agent explores by editing only one file.

The entire architecture, optimizer setup, and training loop live in train.py, so diffs stay focused and the search space stays understandable.

03

Every experiment gets the same clock.

Training is compared under the same five-minute budget, which makes “keep or discard” decisions much more consistent.

Practical notes

          
What to remember before you try it for real.

        

        

          

            
It is not a general-purpose training platform.

            

              Upstream is optimized around a single-GPU, single-file, fixed-budget experiment loop.
              That simplicity is the point.
            

          

          

            
Smaller hardware usually needs retuning.

            

              The README suggests reducing vocabulary size, sequence length, depth,
              eval tokens, and batch size when you move far below the tested H100 setup.
            

          

          

            
Fairness comes from the budget, not identical hardware.

            

              Five minutes makes changes comparable on one machine, but results are still
              not directly portable across every GPU or platform.
